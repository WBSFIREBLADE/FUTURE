# Natural Language to SQL Query Feature

## 🎯 Overview
Your application now supports **natural language queries** on uploaded Excel data. Users can ask questions in plain English, and OpenAI's GPT-4o will generate and execute SQL queries to fetch results.

## 🏗️ Architecture

### Backend Implementation

#### 1. New Functions in `upload/table_manager.py`
- **`get_table_structure(table_name)`**
  - Retrieves the complete schema of a dynamically created table
  - Returns column names and types for OpenAI context

- **`generate_sql_from_prompt(table_name, schema_columns, user_prompt)`**
  - Sends the table schema and user's natural language query to OpenAI GPT-4o
  - Receives and cleans up the generated SQL query
  - Handles markdown code blocks from OpenAI's response

- **`execute_query(sql_query)`**
  - Safely executes SELECT queries only (security check)
  - Returns structured results: columns, rows, row count
  - Error handling for malformed queries

#### 2. New View in `upload/views.py`
- **`QueryTableView`** (POST endpoint: `/api/upload/query/`)
  - **Request payload:**
    ```json
    {
      "table_name": "user_123_filename_...",
      "query_prompt": "Show me all employees with salary > 50000"
    }
    ```
  - **Response:**
    ```json
    {
      "success": true,
      "query_prompt": "Show me all employees with salary > 50000",
      "generated_sql": "SELECT * FROM `user_123_...` WHERE salary > 50000",
      "columns": ["id", "name", "salary", "created_at"],
      "rows": [[1, "John", 60000, "2026-05-26"], ...],
      "row_count": 5
    }
    ```

#### 3. URL Endpoint
- Added to `upload/urls.py`
- Route: `path('query/', QueryTableView.as_view(), name='query')`

### Frontend Implementation

#### Updated `Welcome.jsx` with:
1. **Query Input Interface**
   - Textarea for natural language prompts
   - Submit button for query execution
   - Loading state while generating and executing

2. **Results Table Component**
   - Formatted table displaying query results
   - Column headers with proper styling

3. **Query History**
   - Stores last 10 queries
   - Click history items to re-run queries

4. **Generated SQL Display**

   - Helps users understand query transformations
   - Useful for debugging

## 🔐 Security Features
- ✅ JWT authentication required for all queries
- ✅ Table ownership verification (users can only query their own tables)
- ✅ SELECT-only query execution (prevents INSERT/UPDATE/DELETE)
- ✅ User ID baked into table names
- ✅ Schema isolation per user

## 📝 API Flow Diagram

```
User enters natural language query
           ↓
Frontend sends POST to /api/upload/query/
           ↓
Backend: Get table structure (PRAGMA table_info)
           ↓
Backend: Send schema + prompt to OpenAI GPT-4o
           ↓
OpenAI: Generate SQL query
           ↓
Backend: Clean and validate SQL (SELECT-only)
           ↓
Backend: Execute query on database
           ↓
Backend: Return results in JSON format
           ↓
Frontend: Display in formatted table
```
